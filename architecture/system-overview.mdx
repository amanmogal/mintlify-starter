# System Overview

## Architecture Pattern

AppleSerialScanner follows **MVVM (Model-View-ViewModel)** architecture:

```
┌─────────────────────────────────────────┐
│         Presentation Layer              │
│         (SwiftUI Views)                 │
└─────────────────┬─────────────────────┘
                  │ @Published
                  ▼
┌─────────────────────────────────────────┐
│         ViewModel Layer                  │
│    SerialScannerViewModel                │
└─────┬──────────┬──────────┬─────────────┘
      │          │          │
      ▼          ▼          ▼
┌──────────┐ ┌──────────┐ ┌──────────┐
│ Camera   │ │   OCR    │ │ Services │
│ Manager  │ │Processor │ │          │
└──────────┘ └──────────┘ └──────────┘
```

## Component Layers

### Presentation Layer
SwiftUI views for user interface rendering.

### ViewModel Layer
`SerialScannerViewModel` orchestrates all components and manages state.

### Service Layer
Business logic services for camera, OCR, analytics, validation.

### Utility Layer
Helper classes for validation, configuration, platform detection.
